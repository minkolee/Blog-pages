数据的机器级表示

  1. 实现进制转换
    1. 十进制的25.8125
      25.8125转换成2进制，用整数和分数分别转换。25就是11001。小数部分不断乘以基然后取结果整数。 0.8125*2 = 1.625 1 0.625*2 = 1.25 1 0.25* 2 = 0.5 0 0.5*2 =1 1 25.8125转换成二进制就是 11001.1101
      11001.1101转换成8进制，整数部分补0到 011001，改成8进制就是31，小数部分补0成为.110100，就是.64，最后结果就是31.64
      11001.1101转换成16进制，整数部分补到00011001，就是19，小数部分1101就是D，结果是19.D
    2. 二进制的101101.011
      转成10进制，不断乘以2的各个幂即可。 1*1+1*4+1*8+1*32 = 45，小数部分等于0*1/2 + 1*1/4 + 1*1/8 = 0.375，结果是45.375
      转成8进制，先分组再转换,整数部分是55，小数部分是3，答案是55.3
      转成16进制，同样分组 0010 1101 . 0110,可得结果是2D.6
    3. 8421码的0101 1001 0110.0011
      8421对应的是每位直接展开的10进制表示法，先按4位乘以8421的权重还原成10进制，即596.3
      596的整数转成16进制是254，小数0.3转换成16进制, 先0.3*16 = 4.8，第一位是4，然后0.8*16= 12.8，第二位是12即C，之后尾数还是0.8，所以得到254.4CCCCCC.... 转换成二进制，596转换成二进制是1001010100。小数部分，0.3*2 = 0.6，第一位是0,0.6*2 = 1.2 ，第二位是1， 0.2*2 = 0.4，第三位是0,0.4*2 = 0.8 ，第四位是0， 第五位是0.8*2 = 1.6，是1，然后0.6*2 = 1.2，第六位是1，之后又是0.2开始循环，就又是0011，结果就是1001010100.0100110011.... 先将其分组
    4. 十六进制的4E.C
      这个比较容易，展开成10进制是 4*16+ 14 +12/16 = 78.75
      二进制直接将16进制展开即可，得到 (0)1001110.11
  2. 假定机器数为8位，1位符号，7位数值。写出下列二进制数补码：
    1. +1001，正数的补码是自己，补码就是0001001，再加上符号位0就是00001001
    2. -1001，符号位是1，真值是0001001，补码就是真值取反+1，即 1110110+1 = 1110111，再带上符号位就是11110111
    3. +1，机器数是0000001，补码是00000001
    4. -1，机器数是0000001，补码就是1111110+1 = 1111111，加上符号位就是11111111
    5. +10100，机器数是0010100，补码是00010100
    6. -10100，机器数是0010100，补码是 1101011+1 = 1101100，加上符号就是11101100
    7. +0，补码是00000000
    8. -0，先取二进制表示0000000，然后取反+1 = 1111111+1 = 10000000，最后加上符号位会溢出，还是00000000
  3. 已知补码求原来的表示：其实对符号位为1的补码的数值位再取反加1，就得到真值
    1. 11100111，符号位先去掉，是1100111，减1得到1100110，取反得到0011001，补上符号位是10011001
    2. 10000000这个是最小的负数，其补码，符号位先去掉，是0000000，减1得到1111111，取反得到0000000，补上符号位是10000000
    3. 01010010，这是正数，补码还是01010010
    4. 11010011，去掉符号位，1010011，减1得到1010010，取反得到0101101，加上符号位为10101101

求-26的机器数： -26的二进制是 -10011010，取反加1 得到 1100110 然后补上符号位11100110，也就是E6

-1029的二进制是-10000000101B，补满16位是0000 0100 0000 0101，取15位反再加1 得到 111 1011 1111 1011，再补符号 1111 1011 1111 1011，结果是FBFBH
