集合

  1. 持有对象

集合基础

学完了算法再看这个就有意思多了, 整体分为两大类, Collection 与 Map. 前者是数组类型的实现,比如List和Queue, 后者是符号表.

构造器没有返回值.

this也可以用来返回当前对象的引用, 这个是链式调用的关键, 记得要想到这个方面, 也可以把自身的引用传递给外部方法, 用于类之间的交互, 很有意思.

序列类型

通用的序列接口是Collection, 不过常用的最基础的类型是 ArrayList, 基础的方法是add()添加, get()获取, set(index, T)是设置指定位置为T, 然后返回原来的那个元素.

除了添加单个元素, 还可以添加一组元素, Arrays工具类中有一个asList()方法, 接受一个数组或者用逗号分割的元素列表, 可以转换成一个List对象.

Collection接口里有一个addAll()方法, 接受一个Collection对象, 或者数组和逗号分割的多个参数, 加入到这个Collection中.

Collection的构造器还可以接受另外一个Collection. 但是首选addAll()方法.

Arrays.toString()可以方便快捷打印一个数组.

除了数组实现的ArrayList之外, 还有LinkedList, 是链表实现.

Set

Set本质上也是一种Collection ,Set接口继承字Collection. 其中的值必须不同.

常用的是 HashSet, TreeSet 和 LinkedHashSet
